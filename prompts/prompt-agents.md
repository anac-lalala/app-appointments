# Agents Prompt

Actúa como diseñador de flujo de trabajo con agentes de IA y ayúdame a definir un sistema de agentes claro y coherente para construir este MVP de gestión de citas sin perder consistencia arquitectónica.

Quiero una propuesta:
- clara
- simple
- útil
- orientada a productividad real
- sin hacer que cada agente rediseñe el proyecto
- con buenas fronteras de responsabilidad

---

## Contexto del proyecto

Proyecto MVP de gestión de citas con:
- Next.js frontend
- FastAPI backend
- PostgreSQL
- OTP por email
- JWT
- Docker
- monolito modular
- dominio ya definido
- prompts de arquitectura ya existentes para backend y frontend

Objetivo:
usar agentes de IA como apoyo para diseño, implementación, revisión y documentación, manteniendo coherencia.

---

## Lo que quiero que produzcas

### 1. Principios para usar agentes
Qué reglas deben seguir los agentes para no romper la coherencia del proyecto.

### 2. Roles de agentes recomendados
Propón agentes útiles, por ejemplo:
- arquitecto
- backend implementer
- frontend implementer
- reviewer de calidad
- reviewer de seguridad
- documentador
- planner

### 3. Responsabilidades de cada agente
Qué hace y qué no hace cada uno.

### 4. Inputs y outputs
Qué contexto necesita cada agente y qué debe entregar.

### 5. Reglas de coordinación
Cómo evitar:
- rediseños innecesarios
- contradicciones entre agentes
- duplicación de trabajo
- cambios fuera del alcance del MVP

### 6. Guardrails
Qué restricciones debe respetar cada agente.

### 7. Flujo recomendado de trabajo
Cómo usar estos agentes en secuencia durante el proyecto.

### 8. Criterios de revisión
Qué debe revisar el agente reviewer antes de aceptar cambios.

### 9. Riesgos de usar agentes
Qué errores comunes evitar.

### 10. Propuesta final
Cierra con un sistema de agentes simple y útil para este MVP.

---

## Restricciones

- no demasiados agentes
- no roles redundantes
- no agentes que hagan lo mismo
- sí responsabilidades claras y complementarias

---

## Resultado esperado

La respuesta debe terminar con:
1. lista final de agentes
2. responsabilidades por agente
3. flujo recomendado de uso
4. reglas para mantener coherencia del proyecto
